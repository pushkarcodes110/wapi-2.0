Overview

The **Templates Library** is the centralized repository for pre-approved WhatsApp message templates (Highly Structured Messages or HSM). Within the WAPI ecosystem, templates are essential for proactive communication, enabling administrators to reach out to users with consistent, compliant, and personalized messaging.

Because WhatsApp requires all business-initiated messages to be approved before use, this library serves as your primary workflow zone for ensuring messaging consistency, maintaining compliance with Meta's policies, and enabling rapid, scalable communication across your entire user base.

Key Features

  • Standardized Compliance: Create templates that strictly adhere to WhatsApp's structured messaging protocols.
  • Cross-User Accessibility: Admin-defined templates are accessible across the platform, ensuring unified communication standards.
  • Interactive Payload Support: Integrate rich-text headers, media assets (images, videos), and interactive buttons (CTA, Quick Replies).
  • Dynamic Personalization: Utilize double-bracket placeholders (e.g., `{{1}}`, `{{2}}`) to inject user-specific data in real-time.
  • Status Monitoring: Track approval lifecycles directly from the administration dashboard (Pending → Approved → Rejected).

Navigation Flow

Log in to the WAPI Admin Panel with provisioning authority.
From the sidebar, navigate to Templates Library under the Content section.
You will land on the Templates List Page, showing all global and custom templates available in your system.

Templates List Overview

The list view provides a comprehensive breakdown of your messaging inventory:

Metric Functional Description
Template Name The unique technical identifier (lowercase_with_underscores).
Language The target language code (e.g., `en_US`, `es_ES`).
Message Preview A truncated view of the template body and variable mapping.
Sector The business use-case (e.g., Fashion, Health, Education).
WA Category Specific use-case within the selected sector.(Sale offer, Appointment reminder, new Collection).

Template Creation Workflow

Step 1: Initiation
Click the green + Create Template button in the top-right header.

Step 2: Basic Configuration
Field Requirement & Format
Template Name Must be unique, lowercase, and use underscores (e.g., `welcome_user_v1`).
Template Language Select the specific dialect the message is written in.
Sector & Category Assign the industry sector (General, Fashion, etc.) and use-case type.
Step 3: WhatsApp Service Classification

Choosing the correct Meta category is vital for approval speed and cost-per-message logic:

Marketing

Promotions, offers, product announcements, and recurring engagement.

Utility

Transactional updates, order confirmations, reminders, and account alerts.

Authentication

Single-use passcodes (OTP) and identity verification security messages.

Step 4: Interactive Components & Body
  • Headers: Personalize with bold text or eye-catching media (Image, Video, Document).
  • Message Body: The core content (up to 1600 characters). Ensure variables like `{{1}}` are clearly mapped.
  • Footers: Optional disclaimer or brand taglines (max 60 characters).
  • Buttons: Add 'Quick Reply' icons or 'Call-to-Action' links to boost interaction.

Step 5: Finalization
Review the real-time Phone Preview for visual alignment.
Click Save Template to submit for official Meta review.
Messaging Best Practices
  • Variable Context: Always provide sample content for your variables to prevent rejection.
  • Concise Copy: Keep bodies under 500 characters for optimal mobile readability.
  • Compliance First: Avoid sensitive health data or deceptive marketing phrases.
  • Test Before Send: Verify template rendering in the Internal Chat before launching campaigns.
