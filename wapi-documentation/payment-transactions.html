1. Overview

The Payment Transactions module provides a comprehensive history of all financial activities within your account. It allows users to track payments, monitor transaction statuses, and maintain a clear audit trail of subscription-related charges and custom services. This ensures complete transparency and simplifies financial record-keeping for your business.

Transaction History

View all past and present transaction records in a chronological order.

Status Tracking

Monitor real-time updates for payments, identifying successful completions or pending actions.

Easy Reconciliation

Match payments with specific service contexts like appointments or custom orders.


2. Key Features

  • Centralized Ledger: All payments across different contexts and gateways in one place.
  • Granular Context: Differentiate between subscription plans, appointment bookings, and custom services.
  • Gateway Integration: Supports major payment processors for seamless transactions.
  • Advanced Search: Quickly locate transactions using Order IDs or specific filters.
  • Real-time Updates: Instant feedback on payment success or pending statuses.

3. Interface Breakdown

The Payment Transactions screen is designed for clarity and efficiency, featuring a clean layout that highlights essential financial data.

Top Utility Bar

Contains the search field and filter options to narrow down the transaction list based on your needs.

Main Transaction Table

The core area where all transaction details are displayed in a structured, tabular format.

4. Table Columns Explanation

Understand the data points captured for every transaction record.

Column Description
Order ID A unique alphanumeric identifier assigned to each transaction for tracking purposes.
Payment Link A direct link to the payment gateway or invoice associated with the transaction.
Context Defines the source of the payment (e.g., Appointment for bookings or Custom for other services).
Amount The total monetary value of the transaction, displayed with the respective currency symbol.
Status The current state of the payment (e.g., Paid or Pending).
Gateway The payment service provider used for the transaction (e.g., Razorpay, PayPal).
Payment Date The exact date and time when the transaction was initiated or processed.

5. Actions & Usage

How to effectively manage your transactions on this screen:

  • View Details: Hover or click on a record to see more specific metadata about the payment.
  • Copy Order ID: Use the unique ID for support inquiries or internal bookkeeping.
  • Access Payment Links: Re-open payment windows for pending transactions if required.
  • Export Records: (If enabled) Download your transaction history for offline analysis.

6. Filters & Search

Locate specific transactions quickly using the built-in search and filtering tools.

  • Search Bar: Enter Order IDs or keywords to find specific records instantly.
  • Status Filter: Toggle between Paid and Pending transactions to focus on specific financial states.

7. Status Definitions

Every transaction is assigned a status to indicate its progress.

Paid

The transaction has been successfully processed, and funds have been received.

Pending

The payment is initiated but not yet concluded. This may require user action or gateway confirmation.

8. Use Cases

Subscription Audit

A business owner reviews their monthly transaction log to verify renewal charges for their WAPI subscription plan.

Appointment Verification

An administrator checks the "Context" column to confirm if a client has paid the deposit for an upcoming consultation booking.

Billing Support

An agent uses the Order ID to quickly locate a transaction and resolve a customer's query regarding their payment status.

Pro Tip & Best Practice

Reconcile Regularly: Make it a habit to check for Pending transactions once a week. Often, these are simple abandonments that can be recovered with a quick follow-up message to the customer, ensuring you don't miss out on revenue!
